2. Policy

* Add people to this policy in by way of group.

* AIM -> create policy -> 정책 -> 정책생성

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "s3:ListAllMyBuckets"
            ],
            "Resource": "arn:aws:s3:::*"
        },
        {
            "Effect": "Allow",
            "Action": [
                "s3:ListBucket",
                "s3:GetBucketLocation",
                "s3:ListBucketMultipartUploads",
                "s3:ListBucketVersions"
            ],
            "Resource": "arn:aws:s3:::simtime-bucket"
        },
        {
            "Effect": "Allow",
            "Action": [
                "s3:*Object*",
                "s3:ListMultipartUploadParts",
                "s3:AbortMultipartUpload"
            ],
            "Resource": "arn:aws:s3:::simtime-bucket/*"
        }
    ]
}
```

### S3에 upload 중 발생 하는 400(Bad Request Error)

https://www.codingforentrepreneurs.com/blog/large-file-uploads-with-amazon-s3-django/

> **[ace](https://www.codingforentrepreneurs.com/u/ace/)**2 years, 6 months ago
>
> For anyone following the guide and video: you are instructed to make sure that your region matches the region in your URL (when your browser is on the AWS console).
>
> Unfortunately, not all regions support signature version2 (http://docs.aws.amazon.com/general/latest/gr/signature-version-2.html).
>
> So you will eventually encounter a 400 error (please use sigV4 or something like that).  http://docs.aws.amazon.com/AmazonS3/latest/dev/UsingAWSSDK.html#specify-signature-version
>
> The two fixes are to either edit the boto configurations or change your region.  We are trying this out now.
>
> We are changing our region because we have spent too much time trying to set: [s3]() use-sigv4 = True in the boto config file.



서울은 Signiture version2 를 지원하지 않기 때문에, sigV4를 사용해야한다.

(초기 region들은 둘다 지원하지만 sigV2보다는 sigV4를 권장한다.)
